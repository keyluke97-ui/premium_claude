## 2026-07-08 작업 내용: 완료 팝업 입금 안내 + 요약 예산 제거

### 배경
- 신청 완료 팝업("두 가지만 확인해주세요")을 닫은 뒤 캠지기가 입금을 한 번에 완료하지 않는 문제 (사용자 제기). 팝업이 이메일·대시보드만 안내하고 입금 정보는 팝업 아래 카드에만 있어 스크롤이 필요했음.

### 완료 팝업 개편 (`CompleteStep.jsx`)
- 제목 "두 가지" → **"세 가지만 확인해주세요"**.
- **1번 블록 = 입금 안내(신규)**: 하나은행 `225-910068-71204` · 예금주 · 입금 금액(VAT 포함, 재신청 할인 시 정가 취소선 → 할인가) + 계좌 복사 버튼. 스크롤 없이 팝업 안에서 바로 입금 가능.
- 순서 **입금 → 이메일 → 대시보드**. 입금 블록만 그린 강조(`rgba(1,223,130,.06)` + 그린 보더), 대시보드 블록은 중립 배경으로 하향 — 강조 중복 제거.
- 하단 문구: "먼저 아래 **입금 안내**를 확인해 주세요" → "**입금이 확인되어야** 협찬이 시작돼요".
- CTA "확인했어요" 유지 (닫으면 최상단 입금 안내 카드로 `scrollTo` — 기존 동작).
- 시트에 `maxHeight: 90vh` + `overflowY: auto`, CTA+하단문구를 `sticky bottom:0` 푸터로 고정 — 3블록으로 길어져 375×667(iPhone SE)에서 CTA가 4px 잘리던 문제 해결.

### 신청 내용 요약 카드
- `예산` 행 제거. `budget`은 이 행에서만 쓰여 `CompleteStep` destructuring에서 함께 제거. **FunnelPage는 무수정** (여분 prop 전달은 무해 — 수정 금지 파일 원칙 준수).

### 검증

### 2026-07-08
- **⚠️ 06-10 기록 정정: "합성 `.click()`·`preview_click`이 React onClick을 못 태운다"는 오진이었다.** 이번 세션 실측: `.click()`으로 복사 버튼을 누르자 300ms 내 `복사됨!`로 바뀜 → onClick 정상 발화. 반면 CTA는 클릭 후에도 팝업이 DOM에 남고 `scrollY` 불변이라 "핸들러 미발화"로 오독했는데, 이후 `preview_screenshot`을 찍자 **팝업이 사라지고 페이지가 최상단으로 이동해 있었음**. 진짜 원인은 **핸들러의 *결과*가 rAF에 묶인 것**: AnimatePresence exit은 애니메이션 완료 후에야 언마운트하고 `scrollTo({behavior:'smooth'})`도 rAF 구동 → 헤드리스에서 스로틀. 단순 리렌더로 끝나는 상태 변경(복사 버튼)은 즉시 반영되므로 둘의 차이가 갈렸다. **교훈: 헤드리스에서 "핸들러가 안 먹는다"고 결론짓기 전에 screenshot으로 프레임을 강제하고 재관찰할 것. exit 애니메이션·스무스 스크롤은 DOM 폴링으로 판정하지 말 것.** (06-10의 "dispatch 시퀀스를 써야 한다"는 우회는 불필요했을 가능성)
- **첫 스크린샷을 "색상 깨짐/레이아웃 버그"로 오독.** 진입 애니메이션 중간에 찍혀 opacity가 낮았고 `getBoundingClientRect`가 y≈35.7px 오프셋(시트가 뷰포트 아래로 삐져나옴)을 보였다. 한 번 더 찍으니 정상. **교훈: 첫 스크린샷은 버리고 두 번째부터 판독. rect도 애니메이션 종료 후 측정.**
