## 실수/배운 점

### 2026-06-09
- **커밋 그래프 reachability만으로 "미머지" 단정 → squash 머지 가능성 간과.** `origin/main..branch`가 3커밋 보였다고 "미머지"라 했으나, 사용자가 "대시보드 이미 배포했다"고 지적. GitHub PR이 squash 머지면 커밋은 안 보여도 변경은 main에 있음. **파일 존재 여부(내용) 기준으로 재검증**해야 정확 (`git ls-tree origin/main`, 파일 단위 diff). 배포 상태 확인 원칙 강화.
- **"deploy coupling" 과잉 경보.** 파트너탭 대시보드 변경을 내 쿠폰 브랜치가 끌고온다고 봤으나, 실제론 partner-v3 PR #1(`b7af30e`)로 **이미 origin/main에 있던 것**. 로컬 main이 stale(`65a3736`, origin보다 3 뒤)이라 `git diff --stat main`이 오해 유발 → **반드시 fetch 후 origin/main 기준**으로 비교.
- **핸드오버 가정 ≠ 실제 git 상태.** 문서는 "feature 브랜치에 커밋 누적"이라 했으나 실제론 브랜치 tip=origin/main(`cf6d6db`), 작업은 전부 uncommitted working tree였음. 커밋/머지 전 `git status`+`log`로 실제 상태부터 확인.

### 2026-05-28
- **"필드가 있을 수도?" 의문에 즉답 대신 실 스키마 조회**: 사용자가 새 필드 추가 안내 후 "이미 비슷한 필드 있지 않냐"고 의문 제기. `describe_table` full로 확인 → `추가 기간 연장`은 콘텐츠 제작 개월 연장용(방문 기간 무관), `⏰ 콘텐츠 제작 기한`은 Created 기준 콘텐츠 마감일. 일수 필드는 정말로 없음을 검증 후 추가. 교훈: 운영자가 의문 제기하면 추측 답변 말고 실 스키마부터 확인 (5-27 교훈 강화).
- **formula 필드 vs 입력 필드 구분**: 사용자가 "포뮬러 필드 있는데 왜 굳이 추가하냐"고 의문. 답변: formula는 결과만 계산하고 입력 인자는 별도 필드에 저장 필요 (`⏰ 콘텐츠 제작 기한`도 `기본 제작 개월수` Number 입력 필드 존재). 교훈: formula 활용해도 신청자 input(일수)은 어딘가에 저장돼야 한다는 점을 명확히 설명.

### 2026-05-27
- **초안에 중복 필드 설계**: 별도 `쿠폰 ⭐/✔/🔥 모집 인원` 필드를 만들 계획이었으나, Airtable 실 스키마 조회 후 기존 크루 필드(⭐️ 모집 희망 인원 등)가 이미 있어 중복임을 발견 → 재사용으로 정정. 교훈: 코드·플랜 작성 전 Airtable 실 스키마부터 조회 (4-28 교훈 재확인).
- **프리뷰 전환 stall ≠ 코드 버그**: Claude_Preview 백그라운드 탭(`document.hidden=true`)에서 framer-motion rAF 스로틀로 `mode="wait"` 전환이 매우 느려 약관 단계 렌더 확인에 오래 걸림(코드는 정상). 교훈: 빌드 통과 + 콘솔 에러 0이 코드 무결성의 더 신뢰할 신호. 전환이 안 보이면 프리뷰 환경부터 의심.
- **두 폴더 = 한 repo**: camfit-premium / premium_dashboard는 같은 repo `premium_claude`의 다른 체크아웃. 한쪽만 고치고 끝내면 안 되고 fresh origin/main에서 작업해야 함.

### 2026-04-21
- **로컬 검증 시 submit API 부재로 앱 리셋 현상**: `/api/submit`이 로컬에 없어 신청 제출이 실패하며 FunnelPage가 초기화됨. 이번엔 `fetch` 몽키패칭으로 mock 응답 주입해 CompleteStep 도달. 다음 E2E 검증 시 로컬용 submit stub 혹은 dev 전용 스킵 파라미터 고려.

- **`hasCopied` 상태 리팩터링**: 기존에는 "계좌 복사 후에만 대시보드 링크 노출" 이라는 간접 UX였으나, 대시보드 존재 인지 강화 목적이면 상시 노출이 맞음. 조건부 노출 패턴이 때론 숨기는 부작용이 있음을 인지.
